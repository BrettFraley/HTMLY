H T M L Y

A database-driven composable architecture for static site generation.

Key Features

  • Database-driven composable architecture
  • Uses SSE (Server-Sent Events) or WebSockets for LIVENess
  • Fast, always current state updates to clients

HTMLY is a SQLite-driven code and content-as-data static site generator. It updates connected clients with partial changes over SSE or WebSockets, ensuring dynamic and efficient updates without full-page reloads.

Principle Idea

HTMLY is an idea and exploratory research project by Brett Fraley and YOWN Software & Media. The project aims to define a spec, various implementations, and the the core features that make up a website or software application driven by HTMLY.

The forseen specification involves the combination of static site generation, server side rendering, and composable blocks of dynamic content stored using SQLite.

On top of defining HTMLY as an actual software spec, we are building HTMLY Desktop that features powerful development tools for using or implementing HTMLY websites and services.

Services? Yes, HTMLY is not just for static or dynamic website or applications, it is a general purpose spec for creating all kinds of software tools and web services.

The initial concepts of HTMLY are:

  • All markup, content, and code is in a SQLite database.
  • Websitess, apps, and other software can be built from sets of database schemas and queries.
  • Changes to data triggers content blocks to be regenerated, sent to users, and only the changed content blocks are reloaded.
  • Dynamic content is updated and sent to users via SSE or websockets.

Notes: Jan 17, 2025

We've realized that HTMLY has two main stages of research and development. The first is the spec itself as an idea and the second is initial and future implementations of HTMLY driven or inspired applications using different types of stacks (meaning use of different database engines or configurations, programming languages, integrations with web frameworks, and the testing of these implementations).

There is a third major piece to our development, which mentioned above, is HTMLY-Desktop. A variety of considerations have gone into starting development on HTMLY-Desktop, and while NW.js and other multiplatform desktop development tools have been considered, we are initially choosing Kotlin as our development language. NOTE: We want to experiment with using bash, Crystal, Go, and other languages for the fastest watching of database changes, server side rendering and SSE servers, but we realize that this may be left up to users to experiment with themselves and make those decisions based on their preference and requirements.
